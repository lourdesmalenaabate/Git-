Introduccion

GIT es un sistema de control de versiones desarrollado por Linus Torlvalds debido a que necesitaba llevar un registro de los cambios que realizaba en el Kernel de Linux. El mismo funciona como un Grafo Dirigido Aciclico, es decir, que utiliza vertices(nodos) y aristas para definir la relacion que exite entre cada uno de los objetos del repositorio, que solamente apuntan hacia atras.

Para trabajar con GIT debemos tener en cuenta que este se divide en diferentes areas, que vamos a estar utilizando segun la etapa en la que nos encontremos trabajando.

Working Directory

El area de trabajo corresponde a la carpeta donde inicializamos el respositorio, es tambien el lugar donde vamos a crear, editar y eliminar los diferentes archivos y carpetas del proyecto.

Si bien vamos a estar utilizando la consola circunstancialmente, algunos de los comandos que podemos llegar a utilizar en esta area son los siguientes:

  • dir || ls muestra en contenido de la carpeta actual
  • mkdir <name> crea una carpeta con el nombre indicado
  • cd <path> abre el directorio seleccionado
  • git init inicializa un repositorio en la carpeta actual
  • rm -rf .git elimina el repositorio de git

Staging Area (INDEX)

El area de control de cambios es la encargada de identificar los archivos a los que vamos a realizarle seguimiento, tambien es donde se realizan las capturas de codigo (SNAPSHOT).

Aqui estaremos utilizando la linea de comandos con mas frecuencia y generaremos las capturas para poder comparar y reestablecer nuestros archivos, utilizando los siguiente comandos:

  • git status muestra el estado de los archivos con respecto al INDEX
  • git diff compara el contenido del Working directory con el Staging Area
  • git add <file> Agrega los archivos al Area de control de Cambios
  • git restore --staged <file> Reestablecer el archivo a como se encontraba en la ultima captura
  • git rm --cached <file> Elimina el archivo del Staging Area

Repository(Local/Remote)

El almacen de cambios es donde se encuentra las capturas confirmadas, en archivos BLOB (Binary Large Object), a las que podemos acceder en cualquier momento para visualizar los cambios.

Podemos realizar una gestion local o agregar una direccion remota a la cual subir o solicitar el historial de confirmaciones, con algunos de los siguientes comandos:

  • git log muestra el historial de confirmaciones del repositorio
  • git commit confirma las capturas realizadas
  • git blame <file> muestra los autores de cada linea de un archivo
  • git remote add <alias> <URL> agrega un repositorio remoto
  • git fetch descarga el historial de confirmaciones del remotos
  • git pull descargas e integra los cambios del repositorio remoto
  • git push envia al repositorio remoto el historial de confirmaciones
